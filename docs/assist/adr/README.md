# Architecture Decision Records

Use ADRs for decisions that affect architecture, public interfaces,
determinism, portability, or simulation semantics.

File naming:

```text
0001-use-integer-ticks.md
0002-same-time-event-ordering.md
0003-fmi-importer-selection.md
```

Statuses:

- Proposed
- Accepted
- Superseded
- Rejected

Do not rewrite accepted ADR history. Create a new ADR that supersedes an old
decision.
