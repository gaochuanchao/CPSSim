# Supporting Documentation

This directory contains detailed implementation references, architecture
decisions, development records, coding-agent instructions, migration material,
and earlier project guides retained for maintainers and historical
traceability.

For the maintained reader-facing documentation, start with:

- [User Guide](../user/README.md)
- [Developer Guide](../developer/README.md)

## What belongs here

```text
adr/            durable design and semantic decisions
devlog/         dated implementation and validation evidence
guide/          earlier connected guides and compact semantic references
gui/            detailed GUI architecture and migration records
instructions/   stable engineering and documentation policies
modules/        focused module contracts
refine/         implementation plans, prompts, and acceptance material
```

`COMMANDS.md` and `MODULE-INTERACTIONS.md` remain detailed supporting
references.

Documents in this directory may contain older paths or status statements from
the revision in which they were written. The current User and Developer Guides
are authoritative for reader-facing behavior; use Git history and the relevant
test when a historical document and current code differ.
